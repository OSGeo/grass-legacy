DESCRIPTION

The module v.edit allows the user to edit a vector map layer via Command Line Interface (CLI). Very useful feature for scripts.

v.edit works only with "simple" vector features: points, centroids, lines, boundaries. Currently, only 2D region is supported.

Provides editing features' geometry. Attribute data connected to the vector map layer are not modified. For this purpose, use db.* commands.

Vector features can be selected either by internal id's id, category number cats, coordinates coords, bounding box bbox (x1,y1,x2,y2), polygon polygon or 'where' statement (attribute data) where. Selecting features by coordinates is affected by the current 2D resolution. It can be changed using threshold distance thresh. These options are orthogonal, i.e. can be used in various combinations. For example: 

v.edit map=roads tool=select \
  coord=599505,4921010,603389.0625,4918292.1875 \
  thresh=10000 where="label='interstate'"
selects all features (and prints their id's to standard output) covered by two bounding boxes (center at 599505,4921010 and 603389.0625,4918292.1875, size 2*10000) with attribute label='interstate'.

NOTES

If no vector features are selected or the flag -t is used, topology is not build.

This module can be "buggy", still under development. So please feel free to report all bugs or wanted features. Also this manual page should be updated.

USAGE

Feature selection

Vector features can be selected in several ways:

  • ids - using internal (unique) line id's
  • cats - using category numbers
  • coords - using x,y coordinate pairs (center of bounding box, size defined by thresh)
  • bbox - using bounding box
  • polygon - using polygon (at least 3 coordinate pairs have to be set)
  • where - using 'where' statement (attribute data)

Additional parameters for vector feature specification are:

  • layer - layer number (used only with cats or where option)
  • thresh - threshold distance used for selecting vector features by coordinates

Tool description

  • create - Create new (empty) vector map layer (see v.in.ascii -e). Optionally vector features (in GRASS ASCII standard format) can be read from standard input (input=-) or from the text file given by the input option.
  • add - Add new vector features (stored in GRASS ASCII standard format) to existing vector map layer. Features can be read from standard input or from the given text file (input option). If no header is given, the -n flag must be used.
  • delete - Delete selected vector features from existing vector map layer.
  • copy - Make identical copy of selected vector features.
  • move - Move selected features of existing vector map layer relatively to their current location. This tool requires move option. The option defines coordinates of the movement direction.
  • flip - Flip direction of selected vector lines (lines or boundaries).
  • catadd - Add new layer category(ies) to selected vector feature(s). Category can be later used for new database entry.
  • catdel - Delete layer category(ies) of selected vector feature(s).
  • merge - Merges all (at least two) selected vector lines (lines or boundaries). The nearest distance between start and end nodes of both lines is found. The second line from two selected lines can eventually be flipped in it's direction.
  • break - Split (break) given vector line(s) into two lines on location given by coords. Shape of the line is not changed.
  • snap - Snap one vector line to another.
  • connect - Connect selected lines, the first given line is connected to the second one. The second line is broken if necessary. The lines are connected regarding threshold distance (thresh).
  • vertexadd - Add vertex(ces) to the given vector line(s). Location of the new vertex is given by coord option.
  • vertexdel - Remove vertex(ces) specified by coords option.
  • vertexmove - Move vertex(ces) specified by coords option. Direction of the movement is specified by the move option.
  • select - Print comma separated list of selected line ID's. No editing is done.

EXAMPLES

Create new vector map layer

Create new empty vector map layer: 

v.edit tool=create map=vectmap

Create new vector map layer and read data from file 'roads.txt' (i.e. create a copy of vector map layer 'roads'): 

v.out.ascii in=roads format=standard > roads.txt;
v.edit tool=create map=vectmap input=roads.txt
or alternatively 
cat roads.txt | v.edit tool=create map=vectmap input=-

Add new features to existing vector map layer

Add point to the vector map layer (without header): 

echo "P 1 1
 640794 214874
 1 1" | v.edit -n tool=add map=vectmap

Add new features read from standard input: 

v.out.ascii in=railroads format=standard | v.edit tool=add map=vectmap

Select features and print their ID's

Print ID's of selected features, e.g.: 

v.edit map=soils@PERMANENT tool=select \
  bbox=595733.8125,4919781.75,598536.1875,4917396.75
Example with d.vect: 
d.erase;
d.vect roads;
d.vect -i map=roads cats=`v.edit map=roads tool=select \
  coord=592542.89243878,4924766.99622811,603389.0625,4918292.1875 \
  thresh=1000 --q` col=red

Delete selected features from vector map layer

Remove all vector features with category number 1 or 2: 

v.edit tool=delete map=roads cats=1,2

Remove all vector features except for those with category number 1 or 2 (reverse deletion): 

v.edit -r tool=delete map=roads cats=1,2

Remove features with category 1 or 2 located on coordinates 600952.625,4926107 (bounding box based on the current 2D resolution): 

g.region -d;
v.edit tool=delete map=roads cats=1,2 coords=600952.625,4926107

Remove all features with category 1 and 2 covered by two bounding boxes (center coordinates 592542.892,4924766.996 and 603389.062,4918292.187, size 2000 map units): 

v.edit map=roads tool=delete \
  coord=592542.892,4924766.996,603389.062,4918292.187 \
  thresh=1000 cat=1,2

Move features

Move feature (vector point) located on coordinates 602580,4918480 to coordinates 603580,4919480: 

v.edit tool=move map=archsites coord=602580,4918480 th=1e-2 move=1000,1000

Move all features with category 1 1000 map units to the west and 1000 map units to the south. 

v.edit tool=move map=roads cat=1 move=1000,-1000

Move all features defined by bounding box 601530,4921560,602520,4922310 (W,S,E,N) 1000 map units to the east and 1000 map units to the north: 

v.edit tool=move map=roads bbox=601530,4921560,602520,4922310 move=-1000,1000

Flip direction of vector lines

Flip direction of all vector lines: 

v.edit tool=flip map=streams cats=1-9999 type=line

Add / delete layer category number

Add new layer/category 2/1,2/3,2/4,2/5 to features covered by given polygon: 
v.edit tool=catadd map=roads \
  polygon=599877.75,4925088.375,597164.812,4922524.5,601338.562,4920914.625 \
  layer=2 cat=1,3-5
Delete layer/category 1/1, line id 1: 
v.edit tool=catdel map=roads id=1 cats=5

Connect lines

Connect line id 48 to line id 565: 
v.edit map=roads tool=connect id=48,565
Connect line id 48 to line id 565 and line id 60 to line id 50. Maximum threshold distance is 700 map units: 
v.edit map=roads tool=connect id=48,565,60,50 thresh=700

Add vertex

Add new vertex to the line located at 600952,4926107, threshold is set to 1 map unit. Shape of modified line is not changed: 
v.edit tool=vertexadd map=roads coords=600952,4926107 thresh=1

Delete vertices

Delete vertex located at 593191.608,4925684.849 (threshold set to 0.1 map units). Modify only lines with category 1: 
v.edit tool=vertexdel map=roads coord=593191.608,4925684.849 \
  thresh=1-e1 cats=1

Move vertices

Move vertices located at 604441,4921088 (threshold set to 100 map units). Modify only lines with categories 1-10: 
v.edit tool=vertexmove map=roads cats=1-10 coord=604441,4921088 \
  thresh=100 move=1000,1000
